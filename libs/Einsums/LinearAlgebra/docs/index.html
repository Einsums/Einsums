Linear Algebra#

This module contains the user-facing definitions of various linear algebra routines.

See the API reference of this module for more details.

Public API#

Here are some of the public symbols that are available to use.

template<TensorConcept AType>
void sum_square(const AType &A, RemoveComplexT<typename AType::ValueType> *scale, RemoveComplexT<typename AType::ValueType> *sumsq)#

Computes the sum of the squares of the elements of a tensor. This uses the following formula.

\[scale_{out}^2 * sum_{out} = A_1^2 + A_2^2 + \cdots + A_n^2 + scale_{in}^2 * sum_{in}\]

On exit, the value of scale will normally be 1. However, it may be different if the result of the sum would have caused either an overflow or an underflow.

Currently only supports :code:`BasicTensor`s of rank 1.

Parameters:

  • A – The tensor to evaluate.

  • scale – The scale factor. It is both an input and output parameter.

  • sumsq – The starting point and result. It is both an input and output parameter.

Template Parameters:

AType – The type of the tensor A.

template<bool TransA, bool TransB, MatrixConcept AType, MatrixConcept BType, MatrixConcept CType, typename U>
gemm(U alpha, AType const &A, BType const &B, U beta, CType *C)#

Compute the matrix product between two matrices.

Currently supports the following:

For all of these, U, T = float, double, std::complex<float>, std::complex<double>.

For all of these, the rank is always rank 2.

All of the stored types must match, though the prefactor’s type does not. It will be converted to match if it does not.

  • BasicTensor * BasicTensor = BasicTensor

  • BlockTensor * BlockTensor = BlockTensor

  • TiledTensor * TiledTensor = TiledTensor

  • OtherTensor * OtherTensor = OtherTensor

Parameters:

  • alpha – Scale factor for the matrix product.

  • A – The left matrix in the product.

  • B – The right matrix in the product.

  • beta – The scale factor to apply to the C matrix on input.

  • C – The result. If beta is not zero, then the value of C on input will be scaled and added to the result.

Template Parameters:

  • TransA – If true, transpose the A matrix.

  • TransB – If true, transpose the B matrix.

  • AType – The type of the A matrix.

  • BType – The type of the B matrix.

  • CType – The type of the C matrix.

  • U – The type of the scale factors. It will be converted if needed.

template<bool TransA, bool TransB, MatrixConcept AType, MatrixConcept BType, typename U>
auto gemm(U const alpha, AType const &A, BType const &B) -> RemoveViewT<AType>#

Compute the matrix product between two matrices. This is a wrapper around the previous gemm, but instead of returning its result in an output argument, it returns its result as an output value. It supports all the same combinations as the other definition of gemm, but if it is passed a view, it will remove that view.

Parameters:

  • alpha – The scale factor on the matrix product.

  • A – The left matrix.

  • B – The right matrix.

Returns:

The matrix product scaled by alpha.

template<bool TransA, bool TransB, MatrixConcept AType, MatrixConcept BType, MatrixConcept CType>
void symm_gemm(AType const &A, BType const &B, CType *C)#

This function computes \(OP(B)^T OP(A) OP(B) = C\). It supports the same arguments as gemm, since it normally calls gemm in the back.

Parameters:

  • A – The middle tensor.

  • B – The tensor that will be multiplied on either side.

  • C – The output of the operation.

Template Parameters:

  • TransA – Whether to transpose A.

  • TransB – Whether to transpose the second instance of B. The first instance will always be the opposite.

  • AType – The matrix type of A.

  • BType – The matrix type of B.

  • CType – The matrix type of the output.

template<bool TransA, MatrixConcept AType, VectorConcept XType, VectorConcept YType, typename U>
void gemv(U const alpha, AType const &A, XType const &z, U const beta, YType *y)#

Computes the matrix-vector product.

Currently supports the following:

For each of the arguments, U,T = float, double, std::complex<float>, std::complex<double>.

The stored types of each of the tensors must match.

The rank of A is 2 and the rank of X and Y is 1.

  • BasicTensor * BasicTensor = BasicTensor

  • BlockTensor * BasicTensor = BasicTensor

  • TiledTensor * BasicTensor = BasicTensor

  • TiledTensor * TiledTensor = BasicTensor

  • TiledTensor * BasicTensor = TiledTensor

  • TiledTensor * TiledTensor = TiledTensor

  • OtherTensor * OtherTensor = OtherTensor

Parameters:

  • alpha – The scale factor on the product.

  • A – The matrix in the product.

  • z – The vector in the product.

  • beta – The scale factor on the result vector.

  • y – The result vector. If beta is not zero, then the value of this on entry will be scaled and added to the result.

Template Parameters:

  • TransA – Whether to transpose the matrix.

  • AType – The type of the matrix.

  • XType – The type of the input vector.

  • YType – The type of the output vector.

  • U – The type of the scale factors. If it is not the same as the types stored by the tensors, it will be cast to match.

template<bool ComputeEigenvectors = true, MatrixConcept AType, VectorConcept WType>
void syev(AType *A, WType *W)#

Computes the eigendecomposition of a symmetrix matrix.

Supports the following:

A and W need to have the same stored type, and that type needs to be real.

A needs to be rank 2 and W needs to be rank 1.

  • BasicTensor to BasicTensor

  • BlockTensor to BasicTensor

Parameters:

  • A – On entry, it is the matrix to decompose. On exit, it contains the eigenvectors in its columns, if told to compute the eigenvectors.

  • W – On exit, it contains the eigenvalues.

Template Parameters:

  • ComputeEigenvectors – If true, the eigenvectors will overwrite the A matrix.

  • AType – The type of the matrix.

  • WType – The type of the vector.

template<bool ComputeEigenvectors = true, MatrixConcept AType, VectorConcept WType>
void heev(AType *A, WType *W)#

Computes the eigendecomposition of a Hermitian matrix.

Supports the following:

A needs to be complex, and W needs to be real. The types of the components of A need to be the same as the type of the values of W. For instance, std::complex<float> and float.

A needs to be rank 2 and W needs to be rank 1.

  • BasicTensor to BasicTensor

  • BlockTensor to BasicTensor

Parameters:

  • A – On entry, it is the matrix to decompose. On exit, it contains the eigenvectors in its columns, if told to compute the eigenvectors.

  • W – On exit, it contains the eigenvalues.

Template Parameters:

  • ComputeEigenvectors – If true, the eigenvectors will overwrite the A matrix.

  • AType – The type of the matrix.

  • WType – The type of the vector.

template<bool ComputeLeftRightEigenvectors = true, MatrixConcept AType, VectorConcept WType>
void geev(AType *A, WType *W, AType *lvecs, AType *rvecs)#

Compute the eingendecomposition of a general matrix. If a real matrix has a complex eigenvalue, it will always come in a conjugate pair. In this case, the columns of the eigenvector matrix will act as the real and imaginary parts. The first column of the two will be the real part, and the second column will be the imaginary part of the first eigenvector. The imaginary part of the second eigenvector will be the negative of this vector. This only applies to real inputs. If the input is complex, then the eigenvectors will be stored as normal.

Supports the following:

AType needs to be rank2 and W needs to be rank 1.

W needs to store complex values. A can be real or complex. The stored types much match in precision, so std::complex<float> will match either float or std::complex<float>.

  • BasicTensor to BasicTensor

  • BlockTensor to BasicTensor values and BlockTensor vectors

Parameters:

  • A – The matrix to decompose. It will be overwritten on exit.

  • W – The eigenvalues of the matrix.

  • lvecs – If specified, it will contain the left eigenvectors.

  • rvecs – If specified, it will contain the right eigenvectors.

Template Parameters:

  • ComputeLeftrightEigenvectors – If true, the eigenvectors will be computed.

  • AType – The type of the matrix and the vector outputs.

  • WType – The type of the value output.

template<MatrixConcept AType, MatrixConcept BTyep>
int gesv(AType *A, BType *B)#

Solves a system of linear equations.

Parameters:

A – The coefficient matrix. On exit, it contains the upper and lower triangular factors. The

elements of the lower triangular factor are all 1, so they are not stored. :param B: The constant matrix. If this function returns 0, then on exit, this will contain the solutions. :return: If the return value is greater than 0, then the input matrix is singular. If it is less than zero, then the input contains an invalid value, such as infinity. If it is zero, then the system could be solved.

template<TensorConcept AType>
void scale(typename AType::ValueType scale, AType *A)#

Scales a tensor by a scalar value.

Parameters:

  • scale – The scale factor to apply.

  • A – The tensor to scale.

template<MatrixConcept>
void scale_row(size_t row, typename AType::ValueType scale, AType *A)#
template<MatrixConcept>
void scale_column(size_t col, typename AType::ValueType scale, AType *A)#

Scale a row or column of a matrix.

template<MatrixConcept AType>
auto pow(AType const &a, typename AType::ValueType alpha, typename AType::ValueType cutoff = std::numeric_limits<typename AType::ValueType>::epsilon()) -> RemoveViewT<AType>#

Take the matrix power. This is equivalent to diagonalizing the matrix, raising the eigenvalues to the given power, then recombining the matrix.

Parameters:

  • a – The matrix to exponentiate.

  • alpha – The power to raise the matrix to.

  • cutoff – If an eigenvalue is below this parameter after exponentiation, then set it to be zero.

Returns:

The result of raising the matrix to a power.

template<TensorConcept AType, TensorConcept BType>
auto dot(AType const &A, BType const &B) -> BiggestTypeT<typename AType::ValueType, typename BType::ValueType>#

Compute the dot product between two tensors. This form does not conjugate either element if complex.

template<TensorConcept AType, TensorConcept BType>
auto true_dot(AType const &A, BType const &B) -> BiggestTypeT<typename AType::ValueType, typename BType::ValueType>#

Compute the dot product between two tensors. This form conjugates the first parameter if complex.

template<TensorConcept AType, TensorConcept BType, TensorConcept CType>
auto dot(AType const &A, BType const &B, CType const &C) -> BiggestTypeT<typename AType::ValueType, typename BType::ValueType, typename CType::ValueType>#

Computes the dot product between three tensors.

template<TensorConcept XType, TensorConcept YType>
void axpy(typename XType::ValueType alpha, XType const &X, YType *Y)#

Adds two tensors together. The values from the first tensor will be scaled during addition. This is equivalent to \(Y = aX + Y\).

Parameters:

  • alpha – The scale factor for the input tensor.

  • X – The input tensor.

  • Y – The accumulated tensor.

template<TensorConcept XType, TensorConcept YType>
void axpby(typename XType::ValueType alpha, XType const &X, typename XType::ValueType beta, YType *Y)#

Adds two tensors together. The values from the both tensors will be scaled during addition. This is equivalent to \(Y = aX + bY\).

Parameters:

  • alpha – The scale factor for the input tensor.

  • X – The input tensor.

  • beta – The scale factor for the accumulated tensor.

  • Y – The accumulated tensor.

template<MatrixConcept AType, VectorConcept XYType>
void ger(typename AType::ValueType alpha, XYType const &X, XYType const &Y, AType *A)#

Computes the outer product of two vectors and adds it to the output tensor. Equivalent to \(A = a X Y^T + A\).

Parameters:

  • alpha – The amount to scale the outer product.

  • X – The left vector.

  • Y – The right vector.

  • A – The output matrix.

template<MatrixConcept TensorType>
int getrf(TensorType *A, std::vector<blas::int_t> *pivot)#

Computes the LU factorization of a general \(m\) by \(n\) matrix.

Parameters:

A – The matrix to factorize. On exit, it contains the L and U matrices. The diagonal elements of the

L matrix are not stored, since they are all 1. :param pivot: The pivot table. Indicates which rows were swapped. :return: If 0, then the procedure succeded. Otherwise, the procedure failed.

template<MatrixConcept TensorType>
itn getri(TensorType *A, std::vector<blas::int_t> const &pivot)#

Computes the inverse of a matrix using the data obtained from getrf.

template<MatrixConcept TensorType>
void invert(TensorType *A)#

Combines getrf and getri into one function call, calculating the inverse of the matrix.

enum Norm : char#

Allows selecting of the kind of norm to perform.

enumerator Norm::MaxAbs = 'M'#

Use the maximum absolute value of the tensor as the norm.

enumerator Norm::One = '1'#

Use the 1-norm of the matrix, or the maximum column sum.

enumerator Norm::Infinity = 'I'#

Use the infinity norm of the matrix, or the maximum row sum.

enumerator Norm::Frobenius = 'F'#

Use the Frobenius norm of the matrix, or the square root of the sum of squares of the elements.

template<MarixConcept AType>
auto norm(Norm norm_type, AType const &a) -> RemoveComplexT<typename AType::ValueType>#

Computes the norm of a matrix. The norm can be selected by the first argument, and can come from Norm.

template<TensorConcept AType>
auto vec_norm(AType const &a) -> RemoveComplexT<typename AType::ValueType>#

Compute the Euclidean norm of a vector. This is the usual geometric norm.

template<MatrixConcept AType>
auto svd(AType const &_A) -> std::tuple<Tensor<typename AType::ValueType, 2>, Tensor<RemoveComplexT<typename AType::ValueType>, 1>, Tensor<typename AType::ValueType, 2>>#

Compute the singular value decomposition of a matrix. The order of the elements in the returned tuple is the unitary matrix is first, the rectangular diagonal matrix is second, and the other unitary matrix, transposed.

template<MatrixConcept AType>
auto svd_nullspace(AType const &_A) -> Tensor<typename AType::ValueType, 2>#

Compute the nullspace of a matrix using singular value decomposition.

enum Vectors : char#

Allows selecting of the vectors to use for singular value decomposition.

enumerator Vectors::All = 'A'#

Compute all vectors.

enumerator Vectors::Some = 'S'#

Computes only some of the vectors. The number computed is the same as the smallest dimension of the input matrix.

enumerator Vectors::Overwrite = 'O'#

Overwrites the input matrix with some of the vectors.

enumerator Vectors::None = 'N'#

None of the vectors are computed.

template<MatrixConcept AType>
auto svd_dd(AType const &_A, Vectors job = Vectors::All) -> std::tuple<Tensor<typename AType::ValueType, 2>, Tensor<RemoveComplexT<typename AType::ValueType>, 1>, Tensor<typename AType::ValueType, 2>>#

Compute the singular value decomposition of a matrix, optionally selecting the vectors to compute.

template<MatrixConcept AType>
auto qr(AType const &_A) -> std::tuple<Tensor<typename AType::ValueType, 2>, Tensor<typename AType::ValueType, 1>>#

Compute the QR decomposition of the input matrix. The output is what is needed to pass into later functions. The upper trapezoid of the tensor in the first part of the tuple is the R matrix.

template<MatrixConcept AType, VectorConcept TauType>
auto q(AType const &qr, TauType const &tau) -> Tensor<typename AType::ValueType, 2>#

Use the data from qr to compute the Q matrix.

template<TensorConcept AType, TensorConcept BType, TensorConcept CType, typename T>
void direct_product(T alpha, AType const &A, BType const &B, T beta, CType *C)#

Compute the direct product. This is essentially the element-wise product between two matrices.

template<MatrixConcept AType>
typename AType::ValueType det(AType const &A)#

Computes the determinant of a matrix.
