Linear Algebra#

This module contains the user-facing definitions of various linear algebra routines.

See the API reference of this module for more details.

Public API#

Here are some of the public symbols that are available to use. More can be found in the API reference.

template<TensorConcept AType>
void sum_square(const AType &A, RemoveComplexT<typename AType::ValueType> *scale, RemoveComplexT<typename AType::ValueType> *sumsq)#

Computes the sum of the squares of the elements of a tensor. This uses the following formula.

\[scale_{out}^2 * sum_{out} = A_1^2 + A_2^2 + \cdots + A_n^2 + scale_{in}^2 * sum_{in}\]

On exit, the value of scale will normally be 1. However, it may be different if the result of the sum would have caused either an overflow or an underflow.

Currently only supports :code:`BasicTensor`s of rank 1.

Parameters:

  • A[in] – The tensor to evaluate.

  • scale[inout] – The scale factor. It is both an input and output parameter.

  • sumsq[inout] – The starting point and result. It is both an input and output parameter.

Template Parameters:

AType – The type of the tensor A.

Added in version 1.0.0.

Changed in version 2.0.0: This function can now handle general-rank tensors, as well as block and tiled tensors.

template<bool TransA, bool TransB, MatrixConcept AType, MatrixConcept BType, MatrixConcept CType, typename U>
gemm(U alpha, AType const &A, BType const &B, U beta, CType *C)#

Compute the matrix product between two matrices.

Currently supports the following:

For all of these, U, T = float, double, std::complex<float>, std::complex<double>.

For all of these, the rank is always rank 2.

All of the stored types must match, though the prefactor’s type does not. It will be converted to match if it does not.

  • BasicTensor * BasicTensor = BasicTensor

  • BlockTensor * BlockTensor = BlockTensor

  • TiledTensor * TiledTensor = TiledTensor

  • OtherTensor * OtherTensor = OtherTensor

Parameters:

  • alpha[in] – Scale factor for the matrix product.

  • A[in] – The left matrix in the product.

  • B[in] – The right matrix in the product.

  • beta[in] – The scale factor to apply to the C matrix on input.

  • C[inout] – The result. If beta is not zero, then the value of C on input will be scaled and added to the result.

Template Parameters:

  • TransA – If true, transpose the A matrix.

  • TransB – If true, transpose the B matrix.

  • AType – The type of the A matrix.

  • BType – The type of the B matrix.

  • CType – The type of the C matrix.

  • U – The type of the scale factors. It will be converted if needed.

Throws:

  • rank_error – If the tensor arguments are not rank-2.

  • tensor_compat_error – If the tensor arguments have incompatible dimensions.

Added in version 1.0.0.

template<bool TransA, bool TransB, MatrixConcept AType, MatrixConcept BType, typename U>
auto gemm(U const alpha, AType const &A, BType const &B) -> RemoveViewT<AType>#

Compute the matrix product between two matrices. This is a wrapper around the previous gemm, but instead of returning its result in an output argument, it returns its result as an output value. It supports all the same combinations as the other definition of gemm, but if it is passed a view, it will remove that view.

Parameters:

  • alpha[in] – The scale factor on the matrix product.

  • A[in] – The left matrix.

  • B[in] – The right matrix.

Template Parameters:

  • TransA – Whether to transpose the A matrix.

  • TransB – Whether to transpose the B matrix.

Throws:

  • rank_error – If the tensor arguments are not rank-2.

  • tensor_compat_error – If the tensor arguments have incompatible dimensions.

Returns:

The matrix product scaled by alpha.

Added in version 1.0.0.

template<bool TransA, bool TransB, MatrixConcept AType, MatrixConcept BType, MatrixConcept CType>
void symm_gemm(AType const &A, BType const &B, CType *C)#

This function computes \(OP(B)^T OP(A) OP(B) = C\). It supports the same arguments as gemm, since it normally calls gemm in the back.

Parameters:

  • A[in] – The middle tensor.

  • B[in] – The tensor that will be multiplied on either side.

  • C[out] – The output of the operation.

Template Parameters:

  • TransA – Whether to transpose A.

  • TransB – Whether to transpose the second instance of B. The first instance will always be the opposite.

  • AType – The matrix type of A.

  • BType – The matrix type of B.

  • CType – The matrix type of the output.

Throws:

  • rank_error – If the tensor arguments are not rank-2.

  • tensor_compat_error – If the tensor arguments have incompatible dimensions.

Added in version 1.0.0.

template<bool TransA, MatrixConcept AType, VectorConcept XType, VectorConcept YType, typename U>
void gemv(U const alpha, AType const &A, XType const &z, U const beta, YType *y)#

Computes the matrix-vector product.

Currently supports the following:

For each of the arguments, U,T = float, double, std::complex<float>, std::complex<double>.

The stored types of each of the tensors must match.

The rank of A is 2 and the rank of X and Y is 1.

  • BasicTensor * BasicTensor = BasicTensor

  • BlockTensor * BasicTensor = BasicTensor

  • TiledTensor * BasicTensor = BasicTensor

  • TiledTensor * TiledTensor = BasicTensor

  • TiledTensor * BasicTensor = TiledTensor

  • TiledTensor * TiledTensor = TiledTensor

  • OtherTensor * OtherTensor = OtherTensor

Parameters:

  • alpha[in] – The scale factor on the product.

  • A[in] – The matrix in the product.

  • z[in] – The vector in the product.

  • beta[in] – The scale factor on the result vector.

  • y[inout] – The result vector. If beta is not zero, then the value of this on entry will be scaled and added to the result.

Template Parameters:

  • TransA – Whether to transpose the matrix.

  • AType – The type of the matrix.

  • XType – The type of the input vector.

  • YType – The type of the output vector.

  • U – The type of the scale factors. If it is not the same as the types stored by the tensors, it will be cast to match.

Throws:

  • rank_error – If the first tensor is not rank-2, or the other tensors are not rank-1.

  • tensor_compat_error – If the tensor arguments have incompatible dimensions.

Added in version 1.0.0.

template<bool ComputeEigenvectors = true, MatrixConcept AType, VectorConcept WType>
void syev(AType *A, WType *W)#

Computes the eigendecomposition of a symmetrix matrix.

Supports the following:

A and W need to have the same stored type, and that type needs to be real.

A needs to be rank 2 and W needs to be rank 1.

  • BasicTensor to BasicTensor

  • BlockTensor to BasicTensor

Parameters:

  • A – On entry, it is the matrix to decompose. On exit, it contains the eigenvectors in its columns, if told to compute the eigenvectors.

  • W – On exit, it contains the eigenvalues.

Template Parameters:

  • ComputeEigenvectors – If true, the eigenvectors will overwrite the A matrix.

  • AType – The type of the matrix.

  • WType – The type of the vector.

Throws:

  • rank_error – If the inputs have the wrong ranks. The A tensor needs to be rank-2 and the W tensor needs to be rank-1.

  • dimension_error – If the matrix input is not square.

  • tensor_compat_error – If the length of the eigenvalue vector does not have the same size as the number of rows in the matrix.

  • std::invalid_argument – If values passed to internal functions were invalid. This is often due to passing uninitialized or

zero-size tensors. :throws std::runtime_error: If the eigenvalue algorithm fails to converge.

Added in version 1.0.0.

template<MatrixConcept AType, VectorConcept WType>
void geev(AType *A, WType *W, AType *lvecs, AType *rvecs)#

Compute the eingendecomposition of a general matrix. If a real matrix has a complex eigenvalue, it will always come in a conjugate pair. In this case, the columns of the eigenvector matrix will act as the real and imaginary parts. The first column of the two will be the real part, and the second column will be the imaginary part of the first eigenvector. The imaginary part of the second eigenvector will be the negative of this vector. This only applies to real inputs. If the input is complex, then the eigenvectors will be stored as normal.

Supports the following:

AType needs to be rank2 and W needs to be rank 1.

W needs to store complex values. A can be real or complex. The stored types much match in precision, so std::complex<float> will match either float or std::complex<float>.

  • BasicTensor to BasicTensor

  • BlockTensor to BasicTensor values and BlockTensor vectors

Parameters:

  • A[inout] – The matrix to decompose. It will be overwritten on exit.

  • W[out] – The eigenvalues of the matrix.

  • lvecs[out] – If specified, it will contain the left eigenvectors.

  • rvecs[out] – If specified, it will contain the right eigenvectors.

Template Parameters:

  • ComputeLeftrightEigenvectors – If true, the eigenvectors will be computed. .. versionremoved:: 2.0.0

  • AType – The type of the matrix and the vector outputs.

  • WType – The type of the value output.

::throws rank_error: If the inputs have the wrong ranks. The A tensor needs to be rank-2 and the W tensor needs to be rank-1. :throws dimension_error: If the matrix input is not square. :throws tensor_compat_error: If the length of the eigenvalue vector does not have the same size as the number of rows in the matrix, or the eigenvector outputs, if not null, do not have the same dimensions as the input. :throws std::invalid_argument: If values passed to internal functions were invalid. This is often due to passing uninitialized or zero-size tensors. :throws std::runtime_error: If the eigenvalue algorithm fails to converge.

Added in version 1.0.0.

Changed in version 2.0.0: The option to compute eigenvectors is no longer a template argument. It is now decided by whether the appropriate parameter is a null pointer or not. For instance, if lvecs is a null pointer then it will not be computed.

template<MatrixConcept AType, TensorConcept BType>
int gesv(AType *A, BType *B)#

Solves a system of linear equations.

Parameters:

A[inout[]] – The coefficient matrix. On exit, it contains the upper and lower triangular factors, even if the return value is greater than zero. The

elements of the lower triangular factor are all 1, so they are not stored. :param B[inout]: The constant matrix. If this function returns 0, then on exit, this will contain the solutions. :return: If the return value is greater than 0, then the input matrix is singular. If it is less than zero, then the input contains an invalid value, such as infinity. If it is zero, then the system could be solved.

Throws:

  • rank_error – If the coefficient matrix is not rank-2 or the result matrix is not rank-1 or rank-2.

  • dimension_error – If the coefficient matrix is not square, or the number of rows of the result matrix is not the same as the number

of rows of the coefficient matrix.

Added in version 1.0.0.

Changed in version 2.0.0: Fixed a bug with how the matrices were being handled. The B tensor is now able to be a vector as well.

template<MatrixConcept AType>
auto pow(AType const &a, typename AType::ValueType alpha, typename AType::ValueType cutoff = std::numeric_limits<typename AType::ValueType>::epsilon()) -> RemoveViewT<AType>#

Take the matrix power. This is equivalent to diagonalizing the matrix, raising the eigenvalues to the given power, then recombining the matrix.

Parameters:

  • a[in] – The matrix to exponentiate.

  • alpha[in] – The power to raise the matrix to.

  • cutoff[in] – If an eigenvalue is below this parameter after exponentiation, then set it to be zero.

Returns:

The result of raising the matrix to a power.

Throws:

  • rank_error – If the inputs have the wrong ranks. The A tensor needs to be rank-2.

  • dimension_error – If the matrix input is not square.

  • std::invalid_argument – If values passed to internal functions were invalid. This is often due to passing uninitialized or

zero-size tensors. :throws std::runtime_error: If the eigenvalue algorithm fails to converge.

Added in version 1.0.0.

template<TensorConcept XType, TensorConcept YType>
void axpy(typename XType::ValueType alpha, XType const &X, YType *Y)#

  • X[in] – The input tensor.

  • Y[out] – The accumulated tensor.

Throws:

  • rank_error – If the tensors do not have the same rank.

  • dimension_error – If the tensors do not have the same shape.

template<MatrixConcept AType, VectorConcept XYType>
void ger(typename AType::ValueType alpha, XYType const &X, XYType const &Y, AType *A)#
template<MatrixConcept AType, VectorConcept XYType>
void gerc(typename AType::ValueType alpha, XYType const &X, XYType const &Y, AType *A)#

Computes the outer product of two vectors and adds it to the output tensor. Equivalent to \(\mathbf{A} = \alpha\mathbf{XY}^T + \mathbf{A}\), or \(\mathbf{A} = \alpha\mathbf{XY}^H + \mathbf{A}\) for gerc.

Parameters:

  • alpha[in] – The amount to scale the outer product.

  • X[in] – The left vector.

  • Y[in] – The right vector.

  • A[out] – The output matrix.

Throws:

  • rank_error – If the X and Y tensors are not rank-1 or the A tensor is not rank-2.

  • tensor_compat_error – If the number of elements in the X vector is not the same as the number of rows in A, or the number of

elements in the Y vector is not the same as the number of columns of A.

Added in version 1.0.0: Added ger.

Added in version 2.0.0: Added gerc.
