Allocator whose maximum size can be restricted by runtime variables.

The maximum size is controlled by the global configuration options. In particular, the option to control the size is --einsums:buffer-size and the string in the configuration mapping is buffer-size with the standard variants, such as BUFFER_SIZE .

Use in dynamic allocations with frequent resizing is discouraged, such as in a vector. Resizing requires the buffer to be reallocated while still being allocated. If the buffers are big when resized, this means that the buffer allocation will fail even though the result would be expected to fall within the range of acceptable sizes. However, when using this to allocate a buffer once, this should be fine.

This allocator follows the C++ standard for allocators, and can be used in templates that use them, such as containers, smart pointers, and more.

Public Functions

inline pointer allocate(size_type n)#

Allocate an array of values.

This does not initialize the values in the array. If given zero, this will return the null pointer.

Parameters:

n – The number of elements in the array.

Throws:

std::runtime_error – When the allocation size is too large or the allocation returns an unexpected null pointer.

Returns:

A pointer to the newly allocated memory.

inline bool reserve(size_type n)#

Reserve a number of elements without allocating.

This function is used when you want to allocate some memory with some other allocator, such as the default allocator, but still want to track and limit the memory being used.

Parameters:

n – The number of elements to reserve.

Returns:

True if the allocation will be successful, false if it will fail.

inline void release(size_type n)#

Release a number of elements without freeing.

This function is used when you want to deallocate some memory with some other allocator, such as the default allocator, but still want to track the memory being used.

Parameters:

n – The number of elements to release.

inline void deallocate(pointer p, size_type n)#

Deallocate a number of elements.

Parameters:

  • p – The pointer to free.

  • n – The number of elements the pointer points to.

inline size_type max_size() const#

Query the maximum number of elements the allocator can accept.

This will return the number of elements as determined by the global buffer size limit.

Returns:

The number of elements specified by the buffer size option.

inline size_type available_size() const#

Query the number of elements the allocator has free.

This will return the number of elements that have not yet been allocated.

Returns:

The number of elements available to allocate.
